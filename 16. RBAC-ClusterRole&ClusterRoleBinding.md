# Cluster Role and ClusterRoleBinding

### To check all cluster level resources 
```kubectl api-resources --namespaced=false```


A **ClusterRole** is a non-namespaced Kubernetes resource that defines a set of permissions that applies across the entire cluster. It is a list of rules that specifies allowed operations (verbs like get, list, create, delete) on different resource types (like pods, nodes, or secrets).

A **ClusterRoleBinding** is a non-namespaced Kubernetes resource that links a ClusterRole (the defined set of cluster-wide permissions) to one or more Subjects (Users, Groups, or ServiceAccounts).


**Real-World Scenario**


- **Role** : The permissions defined in a Role grant access to resources only within one namespace.
- **RoleBinding**: The RoleBinding links the specific access card (Role) to an employee (User/Group/ServiceAccount) for their assigned floor (Namespace).
- **ClusterRole**: The permissions defined in a ClusterRole grant access to resources across all floors (all namespaces) or to the building's infrastructure.
- **ClusterRoleBinding**: The ClusterRoleBinding links the Master Key (ClusterRole) to a subject, granting them permissions everywhere in the building.

**Kubernetes doc** : https://kubernetes.io/docs/reference/access-authn-authz/rbac/

cluserRole.yaml 

```
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  # "namespace" omitted since ClusterRoles are not namespaced
  name: node-reader
rules:
- apiGroups: [""]
  #
  # at the HTTP level, the name of the resource for accessing Secret
  # objects is "secrets"
  resources: ["nodes"]
  verbs: ["get", "watch", "list"]
```

clusterrolebinding.yaml

```
apiVersion: rbac.authorization.k8s.io/v1
# This cluster role binding allows anyone in the "manager" group to read secrets in any namespace.
kind: ClusterRoleBinding
metadata:
  name: read-nodes-global
subjects:
- kind: User
  name: suraj # Name is case sensitive
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: ClusterRole
  name: node-reader
  apiGroup: rbac.authorization.k8s.io
```

kubectl config use-context suraj-context \
kubectl get nodes 

**Imperative way:**
```
kubectl create clusterrole --help \
kubectl create clusterrole node-reader --verb=get,list,watch --resource=nodes \
kubectl create clusterrolebinding node-reader-binding --clusterrole=node-reader --user=suraj
```
