This guide shows how to create a new Kubernetes user authenticated via client certificate, approve their CSR, and configure RBAC permissions.

---

## ğŸ“ Prerequisites

Make sure you are inside your working directory:

```bash
cd /Users/satyammishra/Documents/DevSecOps/Kubernetes/piyush-tech/Day23


Generate a Private Key and CSR
Generate a private key and a certificate signing request (CSR) for user krishna:


# Generate private key
openssl genrsa -out krishna.key 2048

# Generate CSR with CN (Common Name) and O (Organization)
openssl req -new -key krishna.key -out krishna.csr -subj "/CN=krishna/O=dev-team"

This creates two files:

krishna.key
krishna.csr
```

Create a file named csr.yaml:

```
apiVersion: certificates.k8s.io/v1
kind: CertificateSigningRequest
metadata:
  name: krishna
spec:
  request: |
    <BASE64_ENCODED_CONTENT>
  signerName: kubernetes.io/kube-apiserver-client
  usages:
  - client auth
```
Now encode the CSR and insert it into the YAML:

cat krishna.csr | base64 | tr -d '\n'


Copy the output and replace <BASE64_ENCODED_CONTENT> in the YAML above.

# Apply and Approve the CSR

```
# Apply CSR to the cluster
kubectl apply -f csr.yaml

# Check CSR status
kubectl get csr

# Approve the CSR
kubectl certificate approve krishna
```

## Extract the Signed Certificate

kubectl get csr krishna -o jsonpath='{.status.certificate}' | base64 --decode > krishna.cert

You should now have:
krishna.key
krishna.cer




# Role
```
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
- apiGroups: [""] # "" indicates the core API group
  resources: ["pods"]
  verbs: ["get", "watch", "list"]
```

**Role binding :**
```
  apiVersion: rbac.authorization.k8s.io/v1
# This role binding allows "jane" to read pods in the "default" namespace.
# You need to already have a Role named "pod-reader" in that namespace.
kind: RoleBinding
metadata:
  name: read-pods
  namespace: default
subjects:
# You can specify more than one "subject"
- kind: User
  name: krishna # "name" is case sensitive
  apiGroup: rbac.authorization.k8s.io
roleRef:
  # "roleRef" specifies the binding to a Role / ClusterRole
  kind: Role #this must be Role or ClusterRole
  name: pod-reader # this must match the name of the Role or ClusterRole you wish to bind to
  apiGroup: rbac.authorization.k8s.io
```

kubectl auth can-i get pods  ==> yes
